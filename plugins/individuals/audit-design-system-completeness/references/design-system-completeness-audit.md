# Design System Completeness Audit

## Decision Heuristics

Completeness means the seed is usable for generation, not production-ready.
Prioritize required sections, provenance, states, accessibility notes, and open
questions.

## Anti-Pattern

Do not fail a seed merely because values are recommended rather than observed;
fail it when the source/confidence distinction is missing.

## Worked Example

A palette with screenshot-derived hex values passes only if the colors are
marked approximate or medium confidence and include contrast validation notes.

## Hand-Off

Return findings that can be fixed by section skills or by regenerating the seed.
