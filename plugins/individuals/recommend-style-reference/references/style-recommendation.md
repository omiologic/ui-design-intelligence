# Style Recommendation

## Decision Heuristics

Rank style fit by project goal, audience expectations, industry trust needs,
scope, component type, accessibility risks, and how much visual personality the
brief can support.

## Anti-Pattern

Do not recommend a loud style because it is visually distinct if it conflicts
with the product task, audience, or trust context.

## Worked Example

For a dental clinic, recommend `japandi` at site/subtle, `bento-box` at
section/medium for service cards, and `glassmorphism` only at component/subtle
if the brand wants a modern dialog treatment.

## Hand-Off

Hand the chosen style id, scope, intensity, preserve rules, and avoid guidance
to a style application or blend workflow.
