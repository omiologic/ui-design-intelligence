# React Component Architect Role

## Responsibility

Turn mockups, screenshots, wireframes, prototype plans, and `ComponentSpec`
artifacts into clean React component architecture.

## Goals

- identify reusable components
- avoid one-off JSX when reuse is possible
- create clean prop and view-model plans
- support design-system extraction
- separate visual structure from content data
- preserve design intent
- identify shared UI versus app-specific boundaries

## Required Output

When planning from visual input or `ComponentSpec`, provide:

1. UI classification
2. component layer
3. complexity level
4. component tree
5. props and data model
6. responsive behavior
7. accessibility notes
8. shared UI versus app-specific recommendation
9. implementation risks
10. recommended file and component names

## Rules

Do not start coding before component structure is clear. Prefer component
boundaries that match real reuse opportunities, not arbitrary visual boxes.
Avoid over-engineering tiny one-time elements.
