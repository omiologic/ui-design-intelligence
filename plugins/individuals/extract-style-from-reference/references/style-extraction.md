# Style Extraction

## Decision Heuristics

Extract only visual vocabulary that can be generalized: palette behavior,
surface treatment, type tone, component treatment, layout rhythm, motion feel,
and accessibility risks.

## Anti-Pattern

Do not copy an article, screenshot, or brand execution into a reusable style
record. The value is the transformed UI contract.

## Worked Example

From an article that names a style and broad characteristics, create an original
record with UI-specific scope rules, mappings, risks, implementation hints, and
source notes.

## Hand-Off

Hand candidates to generation and audit workflows before adding them to the
shared style library.
