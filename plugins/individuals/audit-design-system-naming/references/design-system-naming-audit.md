# Design System Naming Audit

## Decision Heuristics

Names should explain intent and support reuse. Use component and token
vocabulary before inventing new terms.

## Anti-Pattern

Do not allow visual-only names like `blueButton`, `largeCard`, or `softBox` to
become reusable system terms.

## Worked Example

Rename `blueButton` to `button.primary` and map its color reference to
`color.action.primary`.

## Hand-Off

Return replacement names and note any terms that require a new vocabulary entry.
