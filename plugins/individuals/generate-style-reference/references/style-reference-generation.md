# Style Reference Generation

## Decision Heuristics

Create a style record only when the style can guide reusable UI decisions across
scope, component, design-system, or prototype layers.

## Anti-Pattern

Do not write a mood-board paragraph and call it a style reference. The record
must include concrete mappings, risks, compatibility, and source metadata.

## Worked Example

For `bento-box`, describe modular tile rhythm, card hierarchy, component
variants, hover behavior, accessibility risks around dense grids, and mapping to
card/layout tokens.

## Hand-Off

Hand the completed record to recommendation, application, blend, audit, or
design-system mapping workflows depending on the user request.
