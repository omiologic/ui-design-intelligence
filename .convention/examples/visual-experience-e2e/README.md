# Visual Experience E2E Example

This example shows a capture-backed visual experience study for a motion-heavy
homepage. It uses representative file paths for screenshots and motion frames
without storing binary image fixtures.

## Files

- `capture-manifest.json`: example capture manifest with successful stills,
  motion frames, and failed capture evidence.
- `visual-experience-spec.md`: completed implementation-facing visual
  experience handoff.

## Scope

The example models a homepage with scroll-bound palette shifts, pinned hero
behavior, and a fixed canvas/WebGL-like background layer.
