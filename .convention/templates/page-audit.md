## Summary

State the overall page quality, primary risk, and most important corrective
direction.

## Findings

For each finding, record:

- `id`: kebab-case stable identifier
- `title`: concise issue title
- `severity`: `info`, `low`, `medium`, `high`, or `critical`
- `category`: storytelling, structure, interaction, accessibility, responsive,
  SEO, content, or conversion
- `target`: page area, section, component, or interaction
- `evidence`: observable page evidence
- `impact`: user or business effect
- `recommendation`: concrete corrective action
- `handoff`: optional next skill or workflow

## Output

Return JSON matching `.convention/schemas/page-audit.schema.json`.
